# 课程资料整理器：题目与要求

## 一、题目背景

学习一门课时，电脑里经常会出现很多资料：课件、代码、作业说明、数据文件、截图、通知文档等。时间一长，这些文件容易混在一个文件夹里，找起来很麻烦。

请编写一个 Python 程序，把一个目录中的课程资料按照类型自动整理到不同文件夹中。

这个项目的重点不是“文件整理”本身，而是通过一个真实小工具理解 Python 的作用：

1. Python 可以快速编写自动化脚本。
2. Python 适合处理文件、路径和批量任务。
3. Python 可以用字典、函数、模块把规则组织清楚。
4. Python 可以通过命令行参数让程序变得更灵活。

## 二、学习目标

完成本题后，应当能够说明：

1. `Path` 和普通字符串路径有什么区别。
2. `argparse` 如何接收命令行参数。
3. 为什么要先生成“整理计划”，再真正复制或移动文件。
4. 如何用字典保存“后缀名 -> 分类目录”的规则。
5. 如何把一个程序拆成入口文件、核心逻辑模块、分类规则模块。

## 四、功能要求

### 1. 基本运行方式

程序至少支持下面的命令：

```bash
python main.py --source sample_materials --target organized_materials --dry-run
python main.py --source sample_materials --target organized_materials
```

含义：

- `--source`：原始课程资料所在目录。
- `--target`：整理后的目标目录。
- `--dry-run`：只预览整理计划，不真正复制或移动文件。

### 2. 分类规则

按照文件名关键字和文件后缀分类。

如果文件名中包含下面关键字，优先放入 `homework`：

- `作业`
- `练习`
- `实验`
- `任务`

如果文件名不包含上面的关键字，再按后缀分类：

| 类型 | 目录名 | 后缀示例 |
| --- | --- | --- |
| 课件 | `slides` | `.ppt`、`.pptx`、`.key` |
| 代码 | `code` | `.py`、`.ipynb`、`.c`、`.cpp`、`.java` |
| 数据 | `data` | `.csv`、`.xlsx`、`.json` |
| 文档 | `documents` | `.pdf`、`.doc`、`.docx`、`.txt`、`.md` |
| 图片 | `images` | `.png`、`.jpg`、`.jpeg`、`.gif` |
| 其他 | `others` | 无法识别的后缀 |

注意：关键字规则优先级高于后缀规则。例如 `Python第01讲_作业说明.pdf` 虽然是 `.pdf`，但因为文件名包含“作业”，应该放入 `homework`。

### 3. 安全要求

1. 默认使用复制，不删除原文件。
2. `--dry-run` 模式不能创建目标文件，也不能移动或复制文件。
3. 如果目标目录中已经存在同名文件，不能直接覆盖，应自动改名，例如：

```text
课程通知.txt
课程通知_1.txt
课程通知_2.txt
```

### 4. 报告要求

真实整理后，在目标目录中生成 `整理报告.txt`。

报告中至少包含：

1. 本次执行的是复制还是移动。
2. 整理了多少个文件。
3. 每个文件从哪里来到哪里去。
4. 每一类文件有多少个。

## 五、验收命令

在项目目录中运行：

```bash
python main.py --source sample_materials --target organized_materials --dry-run
```

预期效果：

1. 终端输出整理计划。
2. 不创建 `organized_materials` 目录。

继续运行：

```bash
python main.py --source sample_materials --target organized_materials
```

预期效果：

1. 创建 `organized_materials` 目录。
2. 按类型创建子目录。
3. 示例资料被复制到对应目录。
4. 生成 `organized_materials/整理报告.txt`。

## 六、加分任务

完成基本功能后，可以选择下面任意一项扩展：

1. 增加 `--mode move`，支持真正移动文件。
2. 增加 `--recursive`，支持递归整理子目录中的资料。
3. 增加 `--include .py .ipynb`，只整理指定后缀。
4. 增加 `--exclude .png`，排除指定后缀。
5. 把整理报告保存为 CSV 格式。
